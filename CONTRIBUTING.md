# Contributing to OpenQual

Thanks for your interest in OpenQual. We're building an open data standard for first responder qualification data, and we welcome contributions from anyone — whether you work in emergency services, build software for it, or just care about getting this right.

## Where we are

OpenQual is in early stages. Right now, we're focused on defining the core data standard: schemas, data models, and calculation logic for qualification tracking. Code and tooling will follow once the foundational standard is solid.

## How to contribute

**Issues** — If you see a problem, have a question, or want to suggest something, open an issue. No special format required.

**Discussions** — For broader topics, ideas, or questions that don't fit neatly into an issue, use GitHub Discussions.

**Pull requests** — Welcome, especially for the data standard itself (schemas, models, calculation logic). If you're planning a large change, open an issue first so we can talk through the approach.

## What's most helpful right now

- Feedback on schema design and data models
- Real-world qualification workflows we should account for
- Edge cases from departments, disciplines, or jurisdictions we haven't considered
- Documentation improvements

## Code of conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold it.
